| Algorithm   | Best Case  | Average Case | Worst Case |
| ----------- | ---------- | ------------ | ---------- |
| Bubble Sort | O(n)       | O(n²)        | O(n²)      |
| Quick Sort  | O(n log n) | O(n log n)   | O(n²)      |


Why is Quick Sort Preferred?

Quick Sort is generally preferred because:

. It is much faster for large datasets.
. Average time complexity is O(n log n).
. It uses divide-and-conquer strategy. 
. It requires less extra memory compared to Merge Sort.
. It performs efficiently in real-world applications.
